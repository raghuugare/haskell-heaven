Functions

Functions are the raison d'être1 of Functional Programming (Duh!). But it is very surprising to see that many people have some subtle mis-conceptions about the idea of a function, and this leads to difficulties later on while reasoning about functional programming ideas. It is therefore wise to spend some time & effort towards getting a clear picture of functions.

Let us consider some familiar and not-so-familiar examples involving functions from various subjects.

Mathematics? Solve for \(x\) when \(ax^{2}+ bx + \ldots \) ??

Consider the famous Quadratic equation :

\( a x^{2} + b x + c = 0 \)

The above equation states that some function of the independent variable \(x\) is equal to the expression \( {\mbox ax^{2} + bx + c }\) and that it evaluates to zero for some (input) values of \(x\) which are called the roots of the equation.
Symbolically, we can write this as follows:

\(f(x) = ax^{2} + bx + c = 0 \)

Given the constants \(a\), \(b\), & \(c\), we say that the above expression is a function of (=dependent upon) the (independent) variable "x". The equation simply gave us a constraint under which we would proceed to derive expressions for the roots for the given constants. In other words, the we wanted to know for what values of \(x\) does the function \(f(x)\) produce the value of \(0\) (zero), which are called the roots or zeroes of the function \(f(x)\).

Ok, how about Physics ?

Or...do you remember the equation from the Physics of Motion describing the final velocity \(v\) of an object with an initial velocity of \(u\) units, moving under a constant acceleration of \( a \) units after a time \( t \) ??

\( v = u + at \)

Yes!

There is a function hidden here too.
Given all things, the final velocity \( v \) is a function of time, and can be written as:

\(v(t) = u + at\)

Note that the final velocity is actually a function of not only the time, but also the other constants like the initial velocity \( u \) and the acceleration \(a\). But usually, we consider only the variables. Here, the time \( t \) is called the independent variable and the final velocity is called the dependent variable. And in general, when scientists want to plot graphs of the relationship between such variables, the dependent variable is plotted on the y-axis and the independent variable is represented along the x-axis. This fits right in, since such a plot is represented as below:

\( y = f (x) \)

\( dependent\_variable = some\_function (independent\_variable) \)

What is a function then?

A function \(f\) is simply a mapping from a set (called the domain) to another set, which is usually called the function's range.

For example, the function represented by the mathematical expression \(x^{2}\) could be a mapping from the set of both positive & negative integers like \(\{\ldots,-3, -2, -1, 0, +1, +2, +3, \ldots\}\) to the set of positive integers that are perfect squares, such as \(\{0, +1, +4, +9, +16, \ldots\}\)

So are functions exclusively for mathematical stuff ??

No!

This is a major myth about functions among people trying to grok them.

Consider the function that maps the name of a country to its capital city. Such a function does not involve any math expressions. But still, it is a perfectly valid example of what mathematicians would call a function. Remember the mapping part of the definition above to remind you of what constitutes a function!

Mathematical notation for a function

In mathematics, a function is denoted as a mapping in symbolic form as below:

\(f: D \rightarrow R\)
where \(D\) is the Domain and \(R\) is the Range of the function \(f\).

In words, we could read this as,

For every member of the set \(D\) given as the input, the function \(f\) provides a unique value that is a member of the set \(R\).
That is, given the same input, we ought to get the same output from the mapping for it to qualify as a proper function.

Example: square

Consider a function defined over the domain of integers which maps its inputs to their corresponding squares.

Suppose we call this function \(square\). We can obtain this "mapping" by simply multiplying any given integer with itself. Hence we can write the following:

\( square(x) = x^{2} \)

Clearly, the function \( square \) provides a mapping (or a way?) from the set of all Integers to the set of Positive Integers that are perfect squares.

2 things are to be noted here.

  1. The function does not square its argument! In fact, a function never does anything to its argument(s). The argument(s) is/are simply used in providing the mapping value as per the function's name. The argument(s) may even be used to derive/calculate the mapping value via a mathematical expression, a table/database look-up, etc., but the key point is that the argument(s) is/are immutable. Examples are \(log, sine, cosine,\) etc.
  2. Every time we give some value for \(x\), we obtain the same value from applying \(square(x)\). In other words, a function provides a unique, one-to-one mapping w.r.t. the elements in its Domain. Note that it is possible that there exist multiple elements in the Domain, which are mapped to the same value in the Range. But given some input, a function always gives the same value.

We can thus denote \(square\) as below:

\( \mathbb{Z} \xrightarrow{square} \mathbb{Z}^{+} \)

Here, \(\mathbb{Z}\) represents the set of Integers. Note how similar the above is to the generic definition of a function \(f \), which is below:

\( Domain \xrightarrow{f} Range \)

Functions in Haskell

Let us now see how Haskell brings the beauty of functions preserving the purity & the rigor of the above concept.

Just see below how the function square is defined in Haskell: 

		
    square:: Int -> Int
    square x = x * x

There are 2 parts to the above function viz., type signature and the actual definition of the function.

The first line is called the type signature of the function. In essence, it gives a high-level description as to what is the Domain & the Range of the function. (sort of like the interface of the function)
But wait!
How can we represent a set in Haskell??

The answer is Types!
A Type is used to represent a set of related values. In the above example, we have Int representing our mathematical set of Integers (\( \mathbb{Z} \)). And Haskell boasts of one of the best type-systems in modern programming languages.
